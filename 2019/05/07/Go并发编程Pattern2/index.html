何为goroutine

  • goroutine是独立执行的函数，通过go关键字声明
  • goroutine拥有自己独立的调用栈，按需自适应地扩张或收缩
  • 非常轻量，可以创建数千数十万个goroutine
  • goroutine不是线程，程序中可能存在一个线程但是有数千个goroutine。相反，goroutine会根据需要动态多路复用到线程上，以保持所有goroutine运行。
  • 可以把goroutine看做一个轻量级的线程

通信

我们之前的例子，关于那个boring的函数。若只是在多个goroutine中单独执行，代表着每个人说着无聊的话，实际上很脱离实际。因为真实世界中，人们需要通信，才进行对话。

Channel

在go中，使用channel结构进行通信，channel在go中是第一级别的类型。在go中，一个channel为两个goroutine建立连接，允许他们相互通信。

对于boring的例子，我们通过一个channel将执行boring的goroutine与主函数建立连接，这样，他们之间便可进行通信。

同步机制

当主函数执行 <-c 的时候，会等待一个值发送。相似地，当boring函数执行c <- value的时候，它会等待接受者准备就绪。发送者和接收者两者都需要就绪，才能进行一次完整的通信，否则我们必须要等待他们就绪。

channel本身其实既提供了通信也提供了同步机制。

带buffer的channel

可以创建带有buffer的channel，就是并非每次就绪都会通知，而是当填满/清空buffer的时候才会进行下一动作。这种机制移除了同步机制。带有buffer的channel对于某些问题可能很重要，但它们更难以推理。下文不再探讨带有buffer的channel。

Go通信方式

在Go语言编程中，不使用共享内存进行通信，也不在通信中去共享内存。

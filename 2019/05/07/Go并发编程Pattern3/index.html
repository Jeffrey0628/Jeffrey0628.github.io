关于select

select提供了另外一种处理多个channel的方法。像是其他语言中的switch语句，但是每一个case对应的是一种通信。

  • 所有的channel都被评估(evaluate)
  • select本身一直会阻塞，直到其中某一个case能够运行
  • 如果多个case可进行，select会伪随机选一个
  • 若没有就绪的channel，执行default子句

select的这种机制类似于Guarded Command Language (GCL)中的if语句。

使用select进行超时处理

select可以做很多事，其中最常用最重要的情景之一是做超时处理。time.After()返回一个channel，因此可以阻塞select，当时间到达时，select会选择其执行，以达到超时控制的效果。
对于之前的boring程序，如果”Joe”每次讲话太慢，则会直接打印”You’re too slow”。

我们也可以换一种思路，将timeout的定义放在for语句外面，为整个对话限制超时时间。这样，无论Joe说过什么，只要超过5s，他就会触发超时channel，输出”You’re too slow.”。
